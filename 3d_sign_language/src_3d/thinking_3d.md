### 一、 如何解决大量无效帧的问题（动态活动区间检测和自适应采样）

1. 小于8帧直接认为太短舍弃
2. 用valid_indices标识有效帧，先找到有手部出现（索引为0）的第一帧作为起始帧，结束帧为最后一个有效帧（-1）
3. 最终有效帧在16-64帧
4. 不足16或者大于64进行线性填充或者线性采样

### 二、如何裁剪出手部关键点并计算BBOX

1. 裁剪时需要知道具体的21个手部关键点，mp返回的是相对位置，lm.x*image_width,lm.y*image_height，转换为具体像素坐标
2. 利用for循环遍历
3. 得到x_coords,y_coords的取值范围，划定手部的裁切范围
4. 增加1.45（测试过不会导致手部残缺）倍扩展系数，S2HAND需要手腕以下一部分手臂信息确认手掌根部朝向，增加系数避免指尖或者手腕边缘缺失

### 三、如何避免手部坐标快速变化导致的动画画面晃动

1. 获取64帧所有手部坐标
2. 计算并集外接矩形（就是计算一下最大公共外接正方形）
3. 用这个固定矩形裁剪64帧
4. 固定正方形，确保背景禁止，手部移动，让手部相对位移轨迹平滑

### 四、如何解决掉帧问题（最近邻补充）

1. 若第 **\$n\$** 帧未检出手部，优先搜索第 **\$n-1\$** 帧或 **\$n+1\$** 帧的 BBox 坐标。
2. 中间帧用线性插值补齐

### 五、S2HAND单手问题(后续需要翻转左手)

1. S2HAND只建议右手，与我们的任务有一定冲突
2. 选择只重建右手，对左手调用 ``cv2.flip(img, 1)``进行镜像处理，做水平翻转，将其伪装为右手，并记录是否翻转，左手后面需要再翻转回来（is_filpped）

### 六、如何由相对坐标转换为原始坐标

1. offset:new_x_min,new_y_min，表示正方形框左上角的原始坐标
2. scale:max_side，表示正方形框的边长
3. original_size:原图分辨率
4. 转换公式：

$$
P_{original} = \frac{P_{cropped}}{IMAGE\_SIZE} \times max\_side + Offset
$$

### 七、高斯平滑

1. 采用一维高斯平滑函数，减少检测框震颤
2. sigma设置为1.7（经过测试效果较好）,增强平滑，但是降低延迟感

### 八、左右手的并行处理（镜像）

1. 开启mp双手检测，每一帧维护bbox_r,bbox_l两个变量，自动将对应bbox增加到左手与右手，得到两个独立的左右手运动轨迹

### 九、S2HAND相关参数的确定

1. N*21 *3（n代表帧数，21即标准的mp手部关键点坐标数量，3即3维坐标）
2. joints_3d:相对手掌根部坐标：手部关节相对位置，将原点设置在手腕，左手这个参数需要x轴取反
3. pose_params:MANO48维姿态参数：3维代表手腕的整体旋转，45维代表15个手指关节旋转（3轴旋转），是必须的48个角度
4. camera_params:scale,translation（能将3维画面还原到224*224裁切图上） scale:手部在镜头前应该放大还是缩小，因为3d手是标准尺寸，但是人手在视频里面可能距离有变化。  translation:3d手在2d画面的偏移量，将3d标准手从中心移到真实位置
5. betas（shape_params）：有10维，影响手型

### 十一、S2HAND模型配置（加载预训练权重）

1. 核心需求配置

* **`self.train_requires = ['joints', 'verts', 'heatmaps', 'lights']`**
  * **含义** ：定义训练阶段模型需要计算和输出的变量。
  * **作用** ：除了 3D 关节（joints）和顶点（verts），还包括用于辅助监督的 2D 热力图（heatmaps）和用于重构的光照参数（lights）。
* **`self.test_requires = ['joints', 'verts']`**
  * **含义** ：定义测试或推理阶段需要的输出。
  * **作用** ：因为现在是做 3D 数据提取，所以只保留了最核心的坐标数据，删除了不必要的渲染计算以提升速度（光照以及用于自监督学习的热力图）。

2. 模型回归逻辑

* **`self.regress_mode = 'mano'`**
  * **含义** ：指定 3D 手部重构的回归模式。
  * **作用** ：'mano' 表示模型会直接通过神经网络预测 MANO 模型所需的 姿态（Pose）和形状（Shape）参数（必须，是重建最重要的参数）
* **`self.use_mean_shape = False`**
  * **含义** ：是否使用 MANO 的平均手型。
  * **作用** ：设为 `False` 表示模型会根据输入的图片实时预测每个人的手部肥瘦（Shape），这样生成的 3D 模型更贴合原图中的人手。
* **`self.use_2d_as_attention = False`**
  * **含义** ：是否使用 2D 热力图作为特征提取的注意力机制。
  * **作用** ：如果开启，模型会先定位 2D 关键点，然后让编码器重点关注这些区域。目前关闭考虑一个到减少大规模数据的计算，另一个考虑到可能存在2D预测不准确的问题导致错误的注意力，我们对关节精度要求没有那么高

3. 渲染与纹理

* **`self.renderer_mode = 'NR'`**
  * **含义** ：渲染器类型选择。
  * **作用** ：'NR' 代表 Neural Renderer。虽然现在在代码里屏蔽了它，但模型初始化时仍需要这个标记来走通逻辑分支。
* **`self.texture_mode = 'surf'`**
  * **含义** ：手部皮肤纹理的建模方式。
  * **作用** ：'surf'（Surface）表示纹理是基于网格表面的，这与 MANO 的顶点分布相匹配。

4. 输入与数据集

* **`self.image_size = 224`**
  * **含义** ：模型输入图像的分辨率。
  * **作用** ：这是 `EfficientNet-B3` 编码器预期的标准尺寸，所有输入的裁剪图都会被 Resize 成 **$224 \times 224$**。
* **`self.train_datasets = ['FreiHand']`**
  * **含义** ：模型对应的预训练数据集背景。
  * **作用** ：不同的数据集有不同的坐标系（如 RHD 或 FreiHand）。选择 'FreiHand' 是为了让模型内部的相机内参和深度对齐逻辑与下载的权重保持一致。

5. 扩展组件

* **`self.use_pose_regressor = False`**
  * **含义** ：是否开启额外的姿态修正回归器。
  * **作用** ：如果开启，会在 MANO 输出后加一个小型网络做微调。目前设为 `False` 以追求最纯粹的 MANO 参数输出,不开启考虑到防止出现破坏MANO模型导致手指出现奇怪弯曲，而且我们的任务对手部精度要求没有那么高

6. 权重路径

* **`self.pretrain_model = "/home/jm802/.../checkpoint.pth"`**
  * **含义** ：预训练权重文件的绝对路径。
  * **作用** ：这是 `load_model` 函数唯一的加载源。
* **`self.pretrain_segmnet = ... = None`**
  * **含义** ：其他辅助模型（分割、纹理、热力图）的独立权重路径。
  * **作用** ：设为 `None` 表示这些功能要么不使用，要么已经包含在主 `checkpoint.pth` 中了。

### 十三、利用预训练权重进行推理

1. 进行排序，防止因为时间错乱导致渲染错误
2. 进行图片堆叠（CNN常见操作：多输入通道），堆叠为(N,3,224,224)
3. 初始化一个单位矩阵作为相机的内参。由于这是在裁剪后的局部坐标系下推理，通常假设相机位于原点且无畸变。
4. 调用predict_singleview函数，require：joint、vert（用于重建参数）
5. 保存以下参数为np：

* `joints`: 21 个手部关键点的 3D 坐标 **$(N, 21, 3)$**。
* `vertices`: 构成手部皮肤的 778 个网格顶点坐标 **$(N, 778, 3)$**。
* `pose` & `shape`: MANO 模型底层的旋转和形状参数，可以用来驱动其他 3D 角色。

6. 将X轴取反，joints形状(N*21 *3)我们这里的方法：前两个参数全部slice，第三个参数第0维即x取反
7. 翻转vertices后的问题：GPU会判断正反面，镜像后可能导致正面被判断为反面导致光照计算错误，**渲染左手时需要交换面皮环绕顺序避免变黑(已解决)**
   ```
   def load_and_fix_faces(is_left=False):
       # 1. 根据左右手加载对应的权重
       pkl_name = "MANO_LEFT.pkl" if is_left else "MANO_RIGHT.pkl"
       pkl_path = f"/home/jm802/sign_language/3d_sign_language/mano_v1_2/models/{pkl_name}"

       with open(pkl_path, 'rb') as f:
           mano_data = pickle.load(f, encoding='latin1')
       faces = mano_data['f']

       # 2. 【核心】如果是左手，必须交换面片索引顺序
       # 这样能修正 X 轴取反导致的法向量反向，解决“变黑”和“透明感”
       if is_left:
           faces = faces[:, [0, 2, 1]] 

       return faces
   ```

### 十四、利用MANO渲染

1. face的作用：将离散的vertices点云连接为一个平面

- 通过Face计算法向量，同时计算表面法向量与与光线方向夹角（小夹角为直射光线亮，大夹角为斜射光线暗）
- 渲染器在Face出现交叉时，计算Face与相机的距离，距离近的渲染出来

2. 相机、灯光参数：
   - 背景为白色，环境光为中等光照
   - 垂直视野范围为60度，仿照常见人类视觉与单反镜头广角配置，角度越大，手部越远，透视变形越厉害；角度越小，越像平面
   - 相机位姿：齐次变换矩阵

     $$
     \left[
     \begin{array}{ccc|c}
     r_{11} & r_{12} & r_{13} & t_x \\
     r_{21} & r_{22} & r_{23} & t_y \\
     r_{31} & r_{32} & r_{33} & t_z \\
     \hline
     0 & 0 & 0 & 1
     \end{array}
     \right]
     $$

     左上角3*3：旋转矩阵，我首先计算第一帧的中心位置

     右上角3*1：表示相机x,y,z坐标，我这里z设置为0.22(这个高度能填满画面中心)

     最后一行1*4：3d投影到2d
   - 主光源强度为12.0（能产生清晰阴影与高光），之前较小的时候太黑了，颜色为白色
   - 将主光源固定在相机上，保证手部常量
   - 渲染时设置为1024*1024的分辨率更清楚
3. 渲染配置

- 将点云与Face组合为mesh(三角形)
- 将mesh绕y轴旋转180度（录制视频时一般是手背朝向相机），MANO默认是手掌朝向相机
- 将手部放在scene中（之前已经配置好参数）

4. 左手交换面皮环绕顺序的方法：在处理数据时已经翻转x轴，此时逆时针的vertices排列顺序变为顺时针（顺序翻转），导致法向量方向翻转，所以要修正法向量方向，防止背部剔除（我选择交换y,z顺序），此时修正了改变的方向（1，2，3->1,3,2->1,2,3）
